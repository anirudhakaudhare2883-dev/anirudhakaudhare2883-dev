## `⟩ ls ./projects --filter=production`

<br/>

<details>
<summary>
  <b>🧬 Multi-Modal Cancer Detection Framework</b> &nbsp;|&nbsp; <code>CNNs · Ensemble ML · SHAP · Grad-CAM</code>
</summary>
<br/>

**Problem:** Single-source cancer diagnostics are prone to high false-negative rates. Clinical teams need auditable AI, not black boxes.

**System:** Hybrid architecture fusing CNN-extracted image embeddings with ensemble classifiers operating on structured clinical records — two evidence streams, one unified risk output.

**Explainability Layer:** SHAP generates per-feature attribution reports. Grad-CAM produces visual heatmaps over medical images, enabling clinical teams to verify exactly what the model attended to.

**Design Principle:** Modular — new imaging modalities and clinical data streams plug in without retraining the core model.

`Python` `CNNs` `Scikit-learn` `LightGBM` `SHAP` `Grad-CAM` `Flask`

<br/>
</details>

**System:** Multi-output prediction engine mapping patient symptom profiles to: disease classification, medication protocols, dietary guidance, workout recommendations, and precautionary measures — all in a single inference pass.

**Deployment:** Flask REST API serving real-time predictions to a web interface with sub-second response under concurrent load.

`Python` `SVM` `Random Forest` `Flask` `MySQL` `REST API`
